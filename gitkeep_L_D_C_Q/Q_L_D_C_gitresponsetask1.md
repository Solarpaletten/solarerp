### Общая Структура

Репозиторий разделен на несколько основных директорий и файлов, которые отвечают за различные аспекты проекта. Вот краткий обзор:

1. **README.md**
   - Основной файл документации, который обычно содержит общую информацию о проекте, инструкции по установке и запуску.

2. **ai-context**
   - Директория с контекстом для AI-инструментов.
     - `file-list.txt`: Список файлов в проекте.
     - `project-structure.txt`: Структура проекта (как этот файл).

3. **app**
   - Основная директория приложения, содержащая компоненты и маршруты.
     - `(auth)`: Компоненты и маршруты для аутентификации.
       - `login`: Логин-страница.
         - `LoginClient.tsx`
         - `page.tsx`
       - `onboarding`: Пост-регистрационная страница.
         - `page.tsx`
       - `signup`: Регистрация.
         - `page.tsx`
     - `(dashboard)`: Компоненты и маршруты для панели управления.
       - `account`: Учетные записи.
         - `companies`: Страница компаний.
       - `company`: Детальная информация о компании.
         - `[companyId]`: Динамический маршрут для отображения информации о конкретной компании.
       - `layout.tsx`: Общий макет панели управления.
     - `api`: API-маршруты.
       - `account`, `auth`, `company`, `health`: Различные эндпоинты API.
     - `globals.css`: Глобальные стили CSS.
     - `layout.tsx`: Общий макет приложения.
     - `page.tsx`: Основная страница приложения.

4. **components**
   - Директория с компонентами пользовательского интерфейса.
     - `clients`, `erp`, `forms`, `layouts`, `products`, `purchases`, `select`, `ui`: Различные компоненты, отвечающие за разные аспекты UI.

5. **config**
   - Конфигурационные файлы для различных модулей.
     - `clients`, `products`, `purchases`, `sales`: Конфигурации столбцов и других параметров для соответствующих модулей.

6. **cookies.txt**
   - Файл с cookie-данными (возможно, используется для тестирования или разработки).

7. **knowledge**
   - Документация по различным аспектам проекта.
     - `accounting-engine.md`, `api-patterns.md`, `architecture.md`, и т.д.: Файлы с описанием различных компонентов и принципов работы системы.

8. **lib**
   - Библиотека вспомогательных функций и сервисов.
     - `accounting`: Модули для учета.
       - `__tests__`: Тесты.
       - `fifoService.ts`, `journalService.ts`, и т.д.: Сервисные функции.
     - `auth`: Модули для аутентификации.
     - `company`: Сервисы для работы с компаниями.
     - `onboarding`: Модули для пост-регистрационного процесса.
     - `prisma.ts`: Конфигурация Prisma.
     - `services`: Общие сервисы.

9. **middleware.ts**
   - Файл middleware для Next.js, который может содержать логику перехвата запросов и ответов.

10. **next-env.d.ts**
    - Типовые объявления для Next.js.

11. **next.config.js**
    - Конфигурационный файл для Next.js.

12. **node_modules**
    - Директория с зависимостями проекта, управляемая пакетным менеджером (pnpm).

13. **package.json**
    - Файл конфигурации npm, содержащий зависимости и скрипты.

14. **pnpm-lock.yaml**
    - Лок-файл для pnpm, фиксирующий версии зависимостей.

15. **pnpm-workspace.yaml**
    - Конфигурационный файл рабочего пространства pnpm.

16. **postcss.config.js**
    - Конфигурационный файл для PostCSS.

17. **prisma**
    - Директория с конфигурацией и миграциями Prisma.
      - `migrations`: История миграций базы данных.
      - `schema.prisma`: Основной файл схемы Prisma.
      - `seed.ts`: Скрипт для заполнения базы данных тестовыми данными.

18. **scripts**
    - Директория с скриптами.
      - `dev.env`: Переменные окружения для разработки.

19. **tailwind.config.js**
    - Конфигурационный файл для Tailwind CSS.

20. **tmp**
    - Временная директория (возможно, используется для временных данных).

21. **tree.txt**
    - Файл с деревом структуры проекта.

22. **tsconfig.json**
    - Конфигурационный файл для TypeScript.

### Доступ к Файловой Структуре

Чтобы получить доступ к этой структуре, вы можете использовать команды в терминале:

1. **Перейти в директорию проекта:**
   ```sh
   cd /path/to/solar-erp
   ```

2. **Просмотреть структуру директорий и файлов:**
   ```sh
   tree -L 2
   ```

3. **Открыть конкретный файл для просмотра или редактирования:**
   ```sh
   code README.md  # Используя VSCode
   cat README.md   # Просмотреть содержимое файла в терминале
   ```

Если у вас есть дополнительные вопросы или нужна более детальная информация по конкретным файлам или директориям, дайте знать!

На основе предоставленной структуры проекта, вот краткий обзор ключевых компонентов:

components/:

Содержит реакт-компоненты.

Мидлвара для Next.js, может содержать глобальные хуки или проверки.
next.config.js:

Конфигурация Next.js (оптимизации, плагины).
prisma/:

Схема базы данных (schema.prisma) и миграции.

scripts/:

Скрипты для разработки (например, dev.env).
