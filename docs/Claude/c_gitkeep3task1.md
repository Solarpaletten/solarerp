# 📘 CLAUDE — WORKFLOW & CONTEXT CANON (READ FIRST)

## 🔹 Назначение

Этот файл определяет, **как Claude работает с проектом SolarERP**, без зависимости от чатов и временной памяти.

Контекст проекта **живёт в файлах**, а не в диалогах.

---

## 🗂️ Типы файлов и их значение

### 1️⃣ `c_git_wey.md`

**Путь, который мы прошли.**
История решений, архитектурных выборов, ключевых поворотов проекта.

👉 Claude **обязан** сначала читать этот файл, чтобы:

* понимать *почему* архитектура именно такая,
* не предлагать отменённые или устаревшие решения.

---

### 2️⃣ `c_gitkeep*_task*.md`

**Актуальное задание.**
Единственный источник правды о том:

* *что делать сейчас*,
* *в каком виде сдавать результат*.

👉 Если есть несколько `task` — **последний по номеру = активный**.

---

### 3️⃣ Принципиально

* Чаты **не считаются источником контекста**
* Если чат новый — Claude **восстанавливает состояние ТОЛЬКО из файлов**

---

## 🔁 Алгоритм работы Claude (обязательный)

1. 📖 Прочитать `c_git_wey.md`
2. 📌 Открыть последний `c_gitkeep*_task*.md`
3. 🔍 Понять:

   * цель задачи,
   * ограничения,
   * формат результата
4. ⚙️ Выполнить задачу
5. 📄 Сформировать отчёт **C=>D**:

   * что сделано,
   * где лежит,
   * что готово к ревью,
   * что требует решения архитектора
